Étape 1 – Table : Country


---

1. Rôle métier

Country représente un pays de référence.

Il sert à :

définir le cadre administratif national pour les écoles et localités

être la référence pour les régions, unités administratives et localités

standardiser la géographie pour toutes les écoles


⚠️ Country est un référentiel global, stable et indépendant des années scolaires.


---

2. Champs proposés

Champ	Type	Obligatoire	Description

id	PK	oui	Identifiant unique
code	string	oui	Code ISO ou code court (ex. GN)
name	string	oui	Nom du pays (Guinée)
description	text	non	Description facultative
created_at	datetime	oui	Audit
updated_at	datetime	oui	Audit
created_by	FK → User	non	Audit
updated_by	FK → User	non	Audit
is_deleted	boolean	oui	Soft delete
deleted_at	datetime	non	Soft delete
deleted_by	FK → User	non	Soft delete



---

3. Contraintes

Unicité

code UNIQUE

name UNIQUE


Contraintes logiques

Un pays ne peut pas être supprimé physiquement si des régions y sont associées



---

4. Logiques métier (règles)

Country est global et partagé entre toutes les écoles

Toute modification doit vérifier les dépendances dans RegionAdministrative

Une fois créé, le pays est stable et rarement modifié



---

5. Ce qu’on ne met PAS dans Country

❌ Années scolaires ❌ Classes ou écoles ❌ Statut actif/inactif ❌ Localités ou quartiers (gérés par tables filles)


---

6. Dépendances

Utilisé par : RegionAdministrative

Ne dépend d’aucune table master



---

7. Validation Étape 0

✅ Table MASTER stable, abstraite et globale ✅ Conforme aux règles globales ✅ Pas de redondance ni de logique temporelle
