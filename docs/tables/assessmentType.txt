Étape 1 – Table : AssessmentType


---

1. Rôle métier

AssessmentType définit le type d’évaluation académique ou interne.

Il sert à :

catégoriser les évaluations (composition, note de cours, participation)

être utilisé dans CycleAssessment, AssessmentExam et StudentScore

standardiser les types d’évaluation entre écoles


⚠️ AssessmentType est un référentiel global, indépendant des écoles et années scolaires.


---

2. Champs proposés

Champ	Type	Obligatoire	Description

id	PK	oui	Identifiant unique
code	string	oui	Code court (COMPO, COURS, PART)
name	string	oui	Nom lisible (Composition, Note de cours, Participation)
description	text	non	Description détaillée
created_at	datetime	oui	Audit
updated_at	datetime	oui	Audit
created_by	FK → User	non	Audit
updated_by	FK → User	non	Audit
is_deleted	boolean	oui	Soft delete
deleted_at	datetime	non	Soft delete
deleted_by	FK → User	non	Soft delete



---

3. Contraintes

Unicité

code UNIQUE

name UNIQUE


Contraintes logiques

code doit rester stable dans le temps

une fois créé, ne pas supprimer physiquement si utilisé dans des évaluations



---

4. Logiques métier (règles)

AssessmentType est global et partagé entre écoles

Toute modification doit vérifier les dépendances dans CycleAssessment et StudentScore

Les états (actif/inactif) sont gérés via CycleAssessment et non ici



---

5. Ce qu’on ne met PAS dans AssessmentType

❌ Référence directe à une école ou année scolaire ❌ Dates de l’évaluation ❌ Professeurs ou classes ❌ Statut actif/inactif


---

6. Dépendances

Utilisé par : CycleAssessment, AssessmentExam, StudentScore

Ne dépend d’aucune table master
