Étape 1 – Table : Cycle


---

1. Rôle métier

Cycle représente un niveau pédagogique général.

Il sert à :

structurer les niveaux scolaires (ex. Maternelle, Primaire, Collège, Lycée)

servir de référence pour Level et SchoolYearCycle

être partagé entre plusieurs écoles


⚠️ Cycle est un référentiel global, indépendant de l’école et de l’année scolaire.


---

2. Champs proposés

Champ	Type	Obligatoire	Description

id	PK	oui	Identifiant unique
code	string	oui	Code court (MAT, PRI, COL, LYC)
name	string	oui	Nom lisible (Maternelle, Primaire…)
has_track	boolean	oui	Indique si le cycle peut avoir des options/tracks
created_at	datetime	oui	Audit
updated_at	datetime	oui	Audit
created_by	FK → User	non	Audit
updated_by	FK → User	non	Audit
is_deleted	boolean	oui	Soft delete
deleted_at	datetime	non	Soft delete
deleted_by	FK → User	non	Soft delete



---

3. Contraintes

Unicité

code UNIQUE

name UNIQUE


Contraintes logiques

has_track = true uniquement si le cycle peut avoir des options (ex. Lycée)

une fois créé, un cycle MASTER ne doit pas être supprimé physiquement



---

4. Logiques métier (règles)

Cycle est global et partagé entre écoles

Les écoles créent des SchoolYearCycle basés sur ce référentiel

Toute modification d’un cycle doit vérifier les dépendances aux niveaux et aux années scolaires



---

5. Ce qu’on ne met PAS dans Cycle

❌ Référence directe à une école ou année scolaire ❌ Statut actif/inactif (géré dans SchoolYearCycle) ❌ Dates pédagogiques ❌ Professeurs ou classes


---

6. Dépendances

Utilisé par : Level, SchoolYearCycle

Ne dépend d’aucune table master
